## 1. 不可破坏的现有基线（任何批次都必须保持）

整改过程中禁止回退以下行为，违反即视为引入回归：

1. **Dify 主输入**：`mr_text`（builder 输出）/ `mr_txt`（仅 `dify_pusher.py` 映射）语义不变，主输入必须为字符串。

4. **Oracle 兼容**：业务代码不得出现 `Column != ""` / `Column == ""` 的散落写法；布尔比较不得散落使用 `.is_(True)` / `== True` / `== 1`，必须统一走兼容 helper。
5. **中继告警部分保存**：`POST /api/config/relay-alert` 必须保持 `exclude_unset` + merge 语义，禁止改回全量替换。
6. **导出审计**：所有导出接口必须调用 `record_export_audit()`。
7. **配置加密**：`password_enc` / `api_key_enc` / `secret_key_enc` 不得明文落盘；生产环境拒绝默认 `SECRET_KEY`。
8. **Vastbase 列名**：`cursor.description` 必须 `.lower()` 后再 zip。

## 3. 批次 1：P0 安全整改

### 3.1 关闭 `/report/{log_id}` 未认证访问

**位置**：`app/routers/report.py:132-133`

**问题**：HTML 报告页无认证，按 `log_id` 枚举即可获取任意患者姓名、住院号、科室、质控结论。同文件 `get_report_data:107` 已有认证，仅 HTML 路径缺失。

**实现要求**：

1. HTML 页面供临床打印/外链，浏览器 `window.open()` 不能自动附带 `Authorization` header。**禁止**把长期 JWT 放入 `?access_token=`，因为会泄露到浏览器历史、反向代理日志、Referer 和截图。
2. 推荐方案：新增短期 **report token**，仅用于打印报告。
   - 新增接口：`POST /api/report/{log_id}/print-token`，依赖 `get_current_user` 或 `require_permission("view_reports")`。
   - 接口先按科室可见性校验 `log_id`，通过后返回 `{ "token": "...", "expires_in": 300 }`。
   - token 绑定：`purpose=report`、`log_id`、`user_id`、`expire_ts`、签名。TTL 建议 5 分钟。
   - 新建 `app/services/report_token.py`，不要直接复用 `alert_token.py`：现有 `alert_token` 绑定的是 `alert_id`，用于医生端 H5，不适合作为 PushLog 报告 token。
   - `/report/{log_id}` 只接受 `?token=...`，验证失败返回 401；验证成功且 token 内 `log_id` 与 path 不一致返回 401。
3. 可选替代方案：废弃直接打开 HTML，站内只使用已认证的 `/api/report/{log_id}/data` 在 SPA 弹窗中渲染并打印。若保留 HTML 打印页，必须采用 report token。
4. 通过 token 校验后，**追加科室范围校验**（防止签发逻辑 bug 或 token 泄露）：
   - 管理员：放行。
   - 普通用户：`PushLog.dept` 必须等于该用户可见科室集合（经 `Department` 表由 `dept_id`/角色科室反查名称）。
   - token 通道：token 已绑定单一 `log_id`，禁止用该 token 访问其他 `log_id`。
5. 前端 `static/scripts/modules/logs.js:393` 的 `window.open('/report/${logId}')` 改为：先调用 `POST /api/report/{logId}/print-token`，再打开 `/report/${logId}?token=${encodeURIComponent(token)}`。
6. `/api/report/{log_id}/data` 已有登录认证，但仍需套同一套可见性过滤，避免登录用户跨科室读 JSON 报告。

**验收标准**：
- 未带任何 token 访问 `/report/{id}` → 401。
- 普通用户无法为他科室 `log_id` 签发 print-token → 404 或 403（推荐 404，避免泄露资源存在性）。
- 管理员可访问任意。
- report token 跨 `log_id` 访问 → 401。
- report token 过期后访问 → 401。
- 现有站内"查看报告"按钮正常打开。

**测试**：新增 `tests/test_report_auth.py`，覆盖：未登录 HTML、未登录签发 token、普通用户本科室签发成功、普通用户他科室签发失败、管理员签发成功、token 正确访问、token 跨 id 失败、token 过期失败、`/api/report/{id}/data` 他科室失败。

### 3.2 `/api/stats/*` 增加认证与科室隔离

**位置**：`app/routers/stats.py`（全文件）、`app/main.py:168`

**问题**：8 个接口全部仅有 `Depends(get_db)`，无认证；`/anomaly-top` 泄露患者维度数据。

**实现要求**：

1. 每个接口显式接收当前用户，不建议只用 router 级 `dependencies`，因为科室过滤需要拿到 `current_user`：
   ```python
   from app.permissions import require_permission

   def stats_summary(
       db: Session = Depends(get_db),
       current_user: User = Depends(require_permission("view_reports")),
   ):
       ...
   ```
   注意：`require_permission` 对 admin 自动放行，无需额外处理。
2. **科室隔离 helper**（新建 `app/services/dept_visibility.py`）：
   ```python
   def is_admin_user(db, user) -> bool:
       ...

   def visible_dept_names(db, user) -> list[str] | None:
       """admin 返回 None 表示全院；普通用户返回可见科室名称列表。"""

   def apply_push_log_visibility(query, user, db, dept_column=PushLog.dept):
       """admin 不加过滤；普通用户按可见科室名称过滤；无可见科室时返回空结果。"""
   ```
   可见科室来源顺序：
   - admin 角色：全院，返回 `None`。
   - `RoleDepartment`：若该角色配置了多个科室，返回这些科室名称。
   - `User.dept_id`：返回用户所属单科室名称。
   - 普通用户无科室：返回空列表，接口应返回空统计，不应默认全院。
   注意 `PushLog.dept` 是名称字符串，必须经 `Department` 表把 `dept_id` 翻成名称再过滤。
3. 每个统计接口在构建 query 后调用 `apply_push_log_visibility`。涉及接口：
   - `/today`、`/summary`、`/daily`、`/dept`、`/severity`、`/monthly`、`/anomaly-top`、`/dimensions`
   对 `summary` 这类多次 `count()` 的接口，先构造一个 `base_q = apply_push_log_visibility(db.query(PushLog), current_user, db)`，再从 `base_q` 派生 `total/success/failed/inconsistency`，不要每个 count 重新写过滤。
4. `/anomaly-top` 的 `group_by=patient` 分支：非 admin 必须过滤本科室；admin 保留全院。
5. `/dept` 非 admin 只返回可见科室行（单科室或 RoleDepartment 多科室），不能暴露其他科室统计。
6. **补 Oracle 方言**（计划遗漏点）：`/monthly` 的 `func.substr(query_date,1,7)` 改为按 dialect 分支：
   - SQLite：`substr(query_date,1,7)`
   - Oracle：`func.to_char(PushLog.query_date, 'YYYY-MM')`（若 `query_date` 为字符串则用 `substr`，需根据实际列类型确认）
   建议封装 `month_expr(db, column)` helper。

**验收标准**：
- 未登录访问任一 `/api/stats/*` → 401。
- 普通科室用户 `/api/stats/dept` 仅见本科室。
- 普通用户 `/api/stats/anomaly-top?group_by=patient` 仅见本科室患者。
- admin 数据与改造前一致。
- `stats.js`、`dashboard.js` 页面正常渲染（前端会带 JWT，无需改）。
- Oracle 模式 `/monthly` 不报错。

**测试**：`tests/test_stats_auth.py`，覆盖未登录、普通用户本科室、普通用户他科室、admin 全量；`test_stats_oracle_monthly.py` 验证 month_expr 方言分支。

## 4. 批次 2：Oracle 兼容统一层

### 4.1 项 5：清理空字符串比较残留

**新建**：`app/utils/sql_compat.py`

```python
from sqlalchemy import or_

def empty_or_null(column):
    """col IS NULL OR col == ''（Oracle 下 '' 即 NULL，等价 col IS NULL）"""
    return or_(column.is_(None), column == "")

def not_empty(column):
    """col IS NOT NULL（Oracle 下空串已是 NULL，IS NOT NULL 即可）"""
    return column.isnot(None)
```

**整改点**（grep 已确认）：
- `app/routers/logs.py:331,498,563`：`audit_type_code == ""` → 用 4.2 的 `audit_type_scope`。

- `app/services/push_log_supersede.py:39,47-49`：`source_record_key == ""`、`audit_type_code == ""` → `empty_or_null`。
- `app/services/feedback_stats.py:223`：`feedback_text != ""` → 删除该 filter，Python 侧过滤空串（group_by 后判断）。

### 4.3 项 7：统一 Boolean 写法

**新建**：`app/utils/sql_compat.py` 追加：

```python
def bool_true(column):
    """Oracle/SQLite 兼容的布尔真值，避免 .is_(True) 生成不兼容 SQL。"""
    return column == True  # noqa: E712

def bool_false(column):
    """Oracle/SQLite 兼容的布尔假值，避免 .is_(False) 生成不兼容 SQL。"""
    return column == False  # noqa: E712
```
> `column.isnot(None)` 不能表示 True，会把 False 也纳入结果，禁止使用。当前项目既有 Oracle 基线要求布尔 SQL 比较使用 `== True/False`，helper 的价值是集中写法并方便后续方言测试。

**整改点**：

## 10. 部署注意

- 服务器 `/opt/med-audit-docker` 非 git repo，采用 `tar + docker cp + docker commit + docker-compose up -d` 热更新；持久化需 `docker commit`。
- 容器内路径：应用 `/app/app/...`，静态 `/app/static/...`。
- 改动 `database.py` 默认权限后，需重启容器触发 `_ensure_default_rbac_permissions` 幂等补齐。
- `SECRET_KEY` 不可变更（否则历史加密配置无法解密）。
- 涉及前端文件的批次，`docker cp` 后需确认 `__pycache__` 清理与浏览器强制刷新（JS 带 `?v=` 缓存破坏）。
