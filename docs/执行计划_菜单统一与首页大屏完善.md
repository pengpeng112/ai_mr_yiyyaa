# 执行计划：基于 4b9dcc5 继续完善菜单统一、首页驾驶舱和前置机告警

## 复核结论

提示词方向正确，但结合当前代码实际情况，有以下必须调整的点：

### 必须调整

1. **RoleMenuInfo 只有 4 个字段**（`id/label/icon/path`）。提示词要求后端 MENU_CATALOG 返回 `group/order/target/hidden/dev_only`，但 `/api/roles/menus/catalog` 和 `/api/menu/all` 都通过 `RoleMenuInfo` 做 `response_model`，如果不改 schema 就会丢字段（FastAPI 默认截断未定义字段）或报 422。**必须先扩展 RoleMenuInfo 再让后端返回新字段**。

2. **提示词要求前端菜单改为 `el-sub-menu` 渲染**，但 Element Plus 的 `el-sub-menu` 需要嵌套 `el-menu-item`，这在折叠侧边栏场景下表现不如当前 `disabled` 分隔行好。`menu-group-item disabled` 方案在当前项目中更简洁稳定。改为 `el-sub-menu` 会导致侧边栏无法折叠到图标模式时正常显示分组标题。**建议保留当前 disabled 分隔行方案**，只让数据来源从硬编码改为后端驱动。

3. **提示词的 `bootstrapApp()` 中调 `loadCurrentMenu()`**，但 `bootstrapApp` 在 `login()` 和 `restoreSession()` 中都调用了。如果 `loadCurrentMenu()` 网络失败，菜单会回退到 `FALLBACK_MENU`，不会阻塞页面加载。但 `loadCurrentMenu` 返回的是用户可见菜单，不含 `debug` 等 dev_only 菜单——这在 admin-only 场景下需要 `MENU_CONFIG` 而非用户分配菜单来决定 dev_only 可见性。**第一版建议：admin 直接返回全部菜单（含 dev_only），非 admin 返回角色分配菜单。**

4. **提示词方案 A 从非 admin 默认菜单移除 `relay-alert-logs`**，但当前代码已经把 `relay-alert-logs` 写入了 `dept_manager` 和 `auditor` 的默认菜单。直接移除即可，但需要同步确认后端接口权限（`require_role("admin")`）不变。

5. **`require_permission` 已存在**，但需要 `Permission` 表有对应记录。方案 B 需要数据库 migration 或 seed。第一版不做方案 B。

6. **`dashboardLoading` 被 `resetDashboardState()` 清空** 的问题确实存在。`resetDashboardState()` 中 `this.dashboardLoading = false` 会覆盖 `loadDashboard()` 中刚设置的 `this.dashboardLoading = true`。

#### 3.2 新增 `/api/patient-qc/relay-alert/summary` 接口

在 `app/routers/patient_qc.py` 中新增：

```python
@router.get("/relay-alert/summary", summary="前置机推送日志统计")
def relay_alert_summary(
    patient_id: Optional[str] = Query(None),
    status: Optional[str] = Query(None),
    viewed_flag: Optional[int] = Query(None),
    db: Session = Depends(get_db),
    _admin=Depends(require_role("admin")),
):
    from app.models import QCRecordAlertLog
    from sqlalchemy import func, case

    q = db.query(QCRecordAlertLog)
    if patient_id:
        q = q.filter(QCRecordAlertLog.patient_id.like(f"%{patient_id}%"))
    if status:
        q = q.filter(QCRecordAlertLog.status == status)
    if viewed_flag is not None:
        q = q.filter(QCRecordAlertLog.viewed_flag == viewed_flag)

    total = q.count()
    success = q.filter(QCRecordAlertLog.status == "success").count()
    failed = q.filter(QCRecordAlertLog.status == "failed").count()
    pending = q.filter(QCRecordAlertLog.status == "pending").count()
    suppressed = q.filter(QCRecordAlertLog.status == "suppressed").count()
    viewed = q.filter(QCRecordAlertLog.viewed_flag == 1).count()
    unviewed = q.filter(QCRecordAlertLog.viewed_flag == 0, QCRecordAlertLog.status != "suppressed").count()

    return {
        "total": total,
        "success": success,
        "failed": failed,
        "pending": pending,
        "suppressed": suppressed,
        "viewed": viewed,
        "unviewed": unviewed,
        "success_rate": round(success * 100 / total, 2) if total else None,
        "view_rate": round(viewed * 100 / total, 2) if total else None,
    }
```
