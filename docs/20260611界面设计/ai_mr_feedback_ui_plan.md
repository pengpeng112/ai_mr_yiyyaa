
# 质控反馈页面紧凑版 UI 设计与前端改造方案

> 页面：质控反馈  
> 目标：只改前端展示层，不修改后端接口和业务逻辑；保留原有列表、分页、查看详情、打印报告、删除等功能。  
> 设计方向：紧凑型反馈处理台，突出“待确认、高风险、未关闭、可批量处理”。

---

## 1. 当前界面问题

从截图看，当前页面基础功能已经完整，但作为高频处理页面仍有优化空间：

1. 顶部 4 个指标卡片偏大，占据较多首屏高度。
2. 表格上方筛选和处理状态不够突出，用户需要直接进入列表处理。
3. 表格行高还可以进一步压缩，一屏显示更多反馈记录。
4. 操作列“查看详情 / 打印报告 / 删除”按钮重复很多，视觉噪音偏大。
5. “待确认 10718”与“近期不一致病例 10718”数值相同，容易让用户混淆，需要在文案上区分。
6. 左侧菜单中“质控反馈”存在多处重复高亮风险，需要菜单去重。
7. 列表缺少右侧选中反馈摘要，用户要频繁点击详情弹窗。
8. 删除按钮颜色较重，建议放到“更多”或保留为小号危险按钮，避免误触。

## 2. 推荐页面定位

```text
质控反馈处理台
```

核心目标：

```text
1. 先处理待确认和高风险反馈
2. 快速查看患者、科室、核查类型、反馈状态
3. 一屏显示更多患者反馈
4. 右侧快速查看选中记录摘要
5. 支持批量确认、批量关闭、导出
```

## 7. 操作列优化

当前每行显示：

```text
查看详情 / 打印报告 / 删除
```

建议紧凑版改为：

```text
详情 / 打印 / 更多
```

“更多”中放：

```text
确认反馈
关闭反馈
提醒医生
删除
复制患者ID
```

若必须保留删除按钮，也建议：

1. 小号按钮；
2. 二次确认；
3. 鼠标悬浮时才明显显示；
4. 不要默认大红块占视觉主导。

## 8. 右侧反馈摘要建议

点击表格行后，右侧显示：

```text
患者基本信息
质控结果
核查类型
风险等级
反馈状态
主要问题摘要
处理记录
快捷操作：详情 / 打印 / 确认 / 关闭
```

好处：

1. 不用频繁弹窗；
2. 列表处理效率更高；
3. 适合连续处理大量反馈。

## 9. 与现有前端逻辑兼容

当前项目已有 `activeMenu`、`handleMenuSelect`、`el-table`、`el-pagination` 等模式。质控反馈页面可沿用现有页面风格：

1. 新增或复用 `activeMenu === 'feedback'`。
2. 保留当前接口请求函数。
3. 保留查看详情、打印报告、删除原函数。
4. 只调整模板结构和 CSS。
5. 如果后端没有某些统计字段，先由当前列表或已有统计值展示，不强依赖后端。

## 10. 本地 AI 执行提示词

```text
请对“质控反馈”页面做紧凑型 UI 优化，只改前端，不修改后端接口和业务逻辑。

要求：
1. 保留原有列表加载、分页、查看详情、打印报告、删除功能。
2. 不要大 Banner，不要大统计卡。
3. 顶部使用 46px 标题操作条，右侧放刷新、批量确认、批量关闭、导出按钮。
4. 指标区改成一行细条，显示：近期不一致、高风险、待确认、已关闭、今日新增、未打印报告。
5. 筛选区保持一行展示：患者ID、姓名、住院号、科室、核查类型、风险等级、反馈状态、日期范围、查询、重置。
6. 表格使用紧凑行高，表头 34px，行高 38px～42px。
7. 核查类型和反馈摘要单行省略，完整内容放 tooltip 或详情。
8. 操作列显示“详情 / 打印 / 更多”，删除放入更多，或者保留小号删除按钮并做二次确认。
9. 增加右侧选中反馈摘要，不影响原详情弹窗。
10. 左侧菜单去重，质控反馈只保留一个入口。
11. 完成后回归测试：查询、重置、详情、打印报告、删除、分页都必须正常。
```

## 当前实现状态（对照检查）

- **符合度**：50%
- **已落地**：列表/分页/详情弹窗/打印报告/删除/看板视图、状态与严重度筛选、科室筛选、审计类型筛选、时间范围、关键词搜索。
- **已确认（2026-06-14）**：整改闭环率等 KPI 可通过新增后端统计接口或前端已有数据计算展示。
- **未落地**：
  1. 顶部仍为 4 个大统计卡，未改为指标细条（含整改闭环率）。
  2. 操作列仍为“查看详情 / 打印报告 / 删除”三按钮，未收敛为“详情 / 打印 / 更多”。
  3. 缺少右侧选中反馈摘要。
  4. 批量确认、批量关闭未实现（仅有批量删除）。
  5. “待确认”与“近期不一致病例”数值相同造成混淆的问题未在 UI 上区分。
- **建议**：按本文档第 4 节结构改造为紧凑版反馈处理台，优先实现指标细条和右侧摘要。
