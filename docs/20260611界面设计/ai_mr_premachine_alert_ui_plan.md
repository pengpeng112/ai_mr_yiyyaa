
# 前置机告警页面紧凑版 UI 设计与前端改造方案

> 页面：前置机告警  
> 目标：只改前端展示层，不修改后端接口和业务逻辑；保留原有刷新、查询、重置、详情、重试、分页等功能。  
> 设计方向：紧凑型告警工作台，突出“失败/未查看/高危/重试”处理效率。

---

## 1. 当前界面问题

> **已确认（2026-06-14）**：前置机告警从“患者质控”页面的 Tab 拆分为独立菜单页面，放入质控业务组。

拆分前的主要问题：

1. 左侧菜单存在重复高亮或重复“前置机告警”入口，影响定位。
2. 作为患者质控页面的 Tab，与标题区层级较弱，不利于快速处理告警。
3. 指标卡片过大，6 个指标占据较多首屏高度。
4. 筛选区可进一步压缩，保持一行展示。
5. 表格整体已经比较紧凑，但列宽和状态标签还可以优化。
6. “发送状态、已查看、严重度”颜色体系可以更一致。
7. 操作列“详情/重试”较散，建议固定宽度并突出失败记录的“重试”。
8. 当前页面应优先服务“快速处理前置机失败和未查看告警”，不建议做大卡片驾驶舱。

---

## 2. 推荐页面定位

```text
前置机告警处理台
```

核心目标：

```text
1. 一眼看到告警总数、失败数、未查看数、医生查看率
2. 一行完成过滤条件
3. 一屏显示更多告警记录
4. 失败记录可以快速重试
5. 未查看记录可以快速提醒医生
6. 点击一行右侧出现告警详情摘要，减少弹窗频繁打开
```
