
# 推送日志页面紧凑版 UI 改造方案

> 目标：保留原有功能和接口，只调整前端布局与样式；重点解决“间距过大、表格显示患者太少、页面不够紧凑”的问题。  
> 适用页面：推送日志 / 数据统计  
> 推荐风格：医疗后台紧凑型工作台，不做大卡片驾驶舱，不做过多留白，优先保证列表信息密度。

---

## 1. 本版设计方向

上一版偏“驾驶舱展示”，适合汇报，但不适合推送日志这种高频检索页面。

本版改为：

```text
紧凑型日志工作台
```

核心目标：

```text
1. 一屏尽量多显示患者记录
2. 筛选区保持一行，不占太多高度
3. 统计信息只做细条，不做大卡片
4. 表格行高压缩到 38px～44px
5. 长文本压缩显示，详情里查看完整内容
6. 操作按钮缩小，更多操作放下拉
```

---

## 2. 页面结构调整

### 2.1 原设计问题

上一版存在：

```text
1. 顶部 Banner 太高
2. 统计卡片太大
3. 卡片间距太大
4. 表格行高偏大
5. 单条患者记录占用空间太多
```

### 2.2 紧凑版结构

建议页面改为：

```text
页面标题条 48px
├─ 左：推送日志
└─ 右：刷新 / 导出 / 批量重推 / 批量删除

Tab 条 40px
├─ 推送日志
└─ 数据统计

紧凑统计条 42px
├─ 总记录 57522
├─ 跳过 11758
├─ 失败 22
├─ 未复核 9203
└─ 数据为空 2555

筛选条 46px
├─ 状态
├─ 在院科室
├─ 姓名
├─ 患者ID
├─ 核查类型
├─ 日期范围
├─ 查询
└─ 重置

跳过原因细条 36px
├─ 已推送未复核 9203 / 78.27%
└─ 检验检查数据为空 2555 / 21.73%

表格主体
└─ 紧凑行高 40px，尽量一屏显示 12～18 行
```

---

## 3. 紧凑版尺寸规范

### 3.1 页面整体

```text
主内容 padding：12px～14px
卡片圆角：8px
卡片间距：8px～10px
顶部标题条高度：48px
Tab 高度：38px～40px
筛选区高度：44px～48px
```

### 3.2 左侧菜单

```text
侧边栏宽度：220px～228px
菜单项高度：34px～36px
一级菜单字体：13px～14px
二级菜单字体：13px
分组标题：12px
```

推送日志页面不建议侧边栏太宽，否则表格可用宽度减少。

### 3.3 表格

```text
表头高度：36px
行高：38px～44px
字体：12px～13px
单元格 padding：6px 8px
操作按钮：mini size
原因列最多显示 1～2 行
```

---

## 4. 表格列优化

为了尽量多显示患者，建议表格列改为：

| 列 | 宽度 | 说明 |
|---|---:|---|
| 选择 | 36 | 多选框 |
| 患者ID | 96 | 单独显示，便于检索 |
| 姓名 | 70 | 短字段 |
| 在院科室 | 110 | 无则显示 -- |
| 出院科室 | 110 | 无则显示 -- |
| 核查类型 | 170 | 标签 + 省略 |
| 状态 | 58 | 标签 |
| 严重度 | 58 | 标签 |
| 重试 | 48 | 数字 |
| 失败/跳过原因 | 自适应 | 单行或双行省略 |
| 时间 | 132 | 推送时间 |
| 操作 | 120 | 详情 + 更多 |

与上一版不同：不再把患者ID和姓名合并，因为合并后行高会变高。紧凑版应优先横向展示，保证更多行可见。

---

## 5. 筛选区优化

筛选区必须一行展示。建议使用小尺寸输入框：

```text
状态 110px
在院科室 110px
患者姓名 100px
患者ID 110px
核查类型 180px
开始日期 120px
结束日期 120px
查询 56px
重置 56px
```

如果屏幕宽度不足，可以横向滚动筛选区，不要把筛选区变成多行。

---

## 6. 统计区优化

上一版大统计卡太占空间，本版改为细条：

```text
总记录 57522 | 跳过 11758 | 失败 22 | 未复核 9203 | 数据为空 2555
```

高度控制在 42px 内。

颜色建议：

```text
总记录：蓝色
跳过：灰蓝或橙色
失败：红色
未复核：橙色
数据为空：灰色
```

---

## 7. 跳过原因统计优化

不要再用大卡片，改成细条：

```text
跳过原因：已推送未复核 9203（78.27%） | 检验检查数据为空 2555（21.73%）
```

右侧可加小进度条，但高度不要超过 30px。

---

## 8. 操作按钮优化

表格操作列建议：

```text
详情 | 更多
```

“更多”下拉包含：

```text
质控报告
弹出提醒
重推
删除
复制患者ID
```

这样避免一行出现多个按钮撑宽。

## 9. CSS 建议

### 9.1 紧凑模式变量

```css
:root {
  --compact-page-padding: 12px;
  --compact-card-radius: 8px;
  --compact-card-gap: 8px;
  --compact-table-font: 12px;
  --compact-table-row-height: 40px;
  --compact-table-cell-padding: 6px 8px;
}
```

### 9.2 表格压缩

```css
.logs-table .el-table__cell {
  padding: 5px 0 !important;
}

.logs-table .cell {
  line-height: 18px;
  font-size: 12px;
}

.reason-compact {
  max-height: 36px;
  line-height: 18px;
  overflow: hidden;
  display: -webkit-box;
  -webkit-line-clamp: 2;
  -webkit-box-orient: vertical;
}
```
