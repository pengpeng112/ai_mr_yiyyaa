
# 系统配置页面紧凑版 UI 设计与前端改造方案

> 页面：系统配置  
> 目标：只改前端展示层，不修改后端接口和业务逻辑；保留 Oracle/PostgreSQL/Dify/科室过滤/推送参数/通知渠道/前置机推送/SQL 配置等现有功能。  
> 设计方向：紧凑型配置管理台，突出“配置分组、测试状态、风险提示、保存/测试操作、SQL 安全”。

---

## 2. 当前界面主要问题

1. 顶部 Tab 太长，状态标签挤在 Tab 内，阅读成本高。
2. 当前 Tab 卡片内部横向很宽，但表单项仍然纵向堆叠，空间利用不足。
3. Oracle 连接配置中“主机、端口、字段映射、SQL”都在同一卡里，层级不清。
4. SQL 文本框很大，容易把保存/测试按钮挤到底部。
5. 保存配置、测试连接按钮位置较低，用户每次都要滚动。
6. 对数据库配置这种高风险内容，缺少“危险操作提示”和“未测试不建议保存”的视觉提醒。
7. PostgreSQL、Dify、通知渠道等配置建议统一状态格式：未配置、已配置未测试、测试通过、测试失败。
8. 页面适合做成“配置工作台”，不适合大表单无分组堆叠。

## 3. 推荐页面定位

```text
系统配置工作台
```

核心目标：

```text
1. 快速知道每类配置是否完整、是否测试通过
2. 配置项按用途分区：连接信息、字段映射、查询 SQL、操作区
3. 保存和测试操作始终清晰可见
4. SQL 配置可读、可折叠、有风险提示
5. 不改变现有接口，不影响已有保存/测试逻辑
```

## 5. Oracle 数据源 Tab 设计建议

### 5.1 分区

```text
连接信息
├─ 主机地址
├─ 端口
├─ 服务名
├─ 用户名
├─ 密码

字段映射
├─ 患者ID字段
├─ 住院次数字段
├─ 患者姓名字段
├─ 科室字段
├─ 住院号字段

查询 SQL
├─ 病历查询 SQL
├─ 科室查询 SQL

操作区
├─ 保存配置
├─ 测试连接
├─ 预览 SQL
├─ 恢复默认
```

### 5.2 布局

建议当前 Oracle Tab 使用双栏：

```text
左侧 72%：连接信息 + 字段映射 + SQL 编辑器
右侧 28%：测试状态 + 配置说明 + 安全提示 + 字段占位说明
```

---

## 6. SQL 编辑器优化

当前 SQL 文本框很长，建议：

1. 默认高度控制在 220px～260px；
2. 增加“展开编辑”按钮；
3. 增加“格式化 SQL”按钮；
4. 增加“复制 SQL”按钮；
5. 增加“恢复默认 SQL”按钮；
6. 明确提示占位符，例如 `:query_date`、`{dept_filter}`；
7. 保存前提示“请先测试连接或预览 SQL”。

---

## 7. Tab 状态标签规范

```text
未配置：红色/灰红
已配置未测试：橙色
测试通过：绿色
测试失败：红色
已禁用：灰色
```

不要把所有状态都写成长文字，可以改为小胶囊：

```text
Oracle 数据源   已配置·未测
Dify 配置       已配置·未测
科室过滤        未配置
```

## 当前实现状态（对照检查）

- **符合度**：55%
- **已落地**：顶部 Tab 状态胶囊（未配置/已配置未测试/测试通过/测试失败）、Oracle/PostgreSQL/Dify/科室过滤/推送参数/通知渠道/前置机推送/运行总览 Tab、保存/测试连接按钮、密码字段 show-password。
- **未落地**：
  1. 缺少顶部配置状态条（Oracle/PostgreSQL/Dify/科室过滤/推送参数/通知渠道/前置机推送）。
  2. Oracle/PostgreSQL 配置仍为单栏长表单，未按双栏布局（左侧表单 72% / 右侧说明 28%）。
  3. SQL 编辑器未加格式化/复制/恢复默认/展开编辑按钮。
  4. 保存/测试按钮未固定在当前配置卡底部，仍需滚动查找。
  5. 连接信息和字段映射未使用紧凑网格。
- **建议**：保留现有接口调用，仅调整布局和增加工具按钮。
