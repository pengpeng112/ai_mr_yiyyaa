
# 质控反馈详情弹窗紧凑版 UI 设计与前端改造方案

> 页面：质控反馈详情弹窗  
> 目标：只改前端弹窗布局与样式，不修改后端接口和业务逻辑；保留现有查看详情、打印报告、删除/关闭、反馈填写、滚动查看等功能。  
> 设计方向：弹窗内容信息量较大，应采用“固定头部 + 结论摘要 + 锚点导航 + 双栏内容 + 固定底部操作”的紧凑结构。

---

## 1. 当前弹窗问题

从截图看，当前“质控反馈详情”弹窗内容很完整，但存在以下问题：

1. 弹窗内卡片过多，纵向堆叠较长，用户需要频繁滚动。
2. 顶部“总体结论、指标卡、本类型核查重点、质控说明、患者信息”等区域高度偏大。
3. 页面右侧滚动条很长，说明详情页内容没有形成清晰导航。
4. 关键结论“通过 / 需关注 / 已通过 / 风险分 / 推送策略”分散在不同卡片中。
5. “科室反馈与闭环处理”在页面下部，用户滚动到底部才看到处理入口。
6. “原始记录与推送文本”等长内容默认占空间，应折叠。
7. 弹窗底部操作按钮在截图中被遮挡或需要滚动，不利于快速处理。
8. 当前弹窗偏“报告展示”，实际工作更需要“快速复核 + 处理闭环”。

---

## 2. 推荐弹窗定位

```text
质控反馈详情处理弹窗
```

目标是：

```text
1. 打开弹窗第一屏看到结论、风险、患者、处理状态
2. 中间区域快速定位核查重点和问题详情
3. 长文本默认折叠，按需展开
4. 反馈处理区始终容易找到
5. 底部操作按钮固定，不随内容滚动消失
```

---

## 3. 推荐整体结构

```text
弹窗宽度：1120px～1180px
最大高度：86vh
内部布局：
┌──────────────────────────────────────────────┐
│ 固定头部：标题 + 患者/核查类型 + 关闭          │
├──────────────────────────────────────────────┤
│ 结论摘要条：通过/风险分/需关注/已通过/推送策略 │
├──────────────────────────────────────────────┤
│ 锚点导航：总览 | 核查重点 | 质控说明 | 对照项 | 原文 | 反馈 │
├──────────────────────────────────────────────┤
│ 内容区：左侧主内容 70% + 右侧患者/处理摘要 30% │
├──────────────────────────────────────────────┤
│ 固定底部：查看报告 / 打印 / 确认 / 关闭 / 删除 │
└──────────────────────────────────────────────┘
```

## 4. 设计细节

### 4.1 固定头部

当前标题只显示“质控反馈详情”，建议增强为：

```text
质控反馈详情
首页手术与首次病程质控反馈
0396480 / 高血压、心衰疾病科 / 入院与首次病程核查
```

头部高度控制在 56px～64px。

### 4.2 结论摘要条

把截图中的“通过、需关注、已通过、核查项、风险分、推送策略、推送时间”合并为一条紧凑摘要区：

```text
总体结论：通过
核查项：5
需关注：0
已通过：5
风险分：10
推送策略：仅复核
推送时间：2026-06-14 13:50:48
```

高度控制在 54px～64px，不再拆成多行大卡片。

### 4.3 锚点导航

增加一行轻量导航：

```text
总览 | 核查重点 | 质控说明 | 手术对照 | 原文记录 | 反馈处理
```

点击后滚动到对应区域。

### 4.4 内容双栏

左侧主内容：

```text
本类型核查重点
质控说明
首页手术与首次病程对照
原始记录与推送文本
科室反馈与闭环处理
```

右侧固定摘要：

```text
患者与就诊信息
反馈状态
维度结果
处理时间线
快捷操作
```

右侧摘要 sticky，滚动主内容时仍可看到患者信息和处理状态。

### 4.5 长内容折叠

以下内容默认折叠：

```text
通过项速览
原始记录与推送文本
文书原文
模型返回 JSON
```

默认只展示摘要，点击展开查看完整内容。

### 4.6 反馈处理区提前

不要把“科室反馈与闭环处理”放在很底部。建议在右侧摘要中放快捷入口，主内容底部保留完整表单。

### 4.7 固定底部操作栏

底部固定：

```text
关闭
打印报告
复制摘要
确认反馈
关闭闭环
删除
```

危险操作“删除”放最右，且必须二次确认。
