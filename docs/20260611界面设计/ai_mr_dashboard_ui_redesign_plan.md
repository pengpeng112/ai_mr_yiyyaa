# AI质控运营驾驶舱前端 UI 重设计与改造计划

> 项目：`pengpeng112/ai_mr_yiyyaa`  
> 页面：AI质控首页 / 首页总览 / 质控驾驶舱  
> 目标：只改前端，尽量不修改后端接口；在保留现有功能的前提下，优化布局、样式、菜单结构、图表可读性与交互体验。  
> 设计方向：医疗信息化、蓝白主色、驾驶舱式总览、可读性优先、低风险逐步替换。

---

## 1. 现状判断

当前页面已经具备完整的监控和运营指标，但视觉层级、菜单治理和信息密度仍需优化。

### 1.1 当前主要问题

1. 左侧菜单过长，并且存在大量重复分组，如“质控闭环、患者质控、前置机告警”等重复出现。
2. 左侧菜单字体偏小，层级区分弱，长时间使用容易疲劳。
3. 首页指标卡片较多，但缺少分组，用户第一眼不容易判断重点。
4. 图表区信息密度较高，趋势图日期标签拥挤，维度分布中文标签较长。
5. 系统健康矩阵可读性一般，状态优先级不够明显。
6. 中下部卡片存在大片空白，空状态表现较弱。
7. 前置机闭环适合做成流程/漏斗，而不是普通列表。
8. 页面整体“能看数据”，但还不像成熟的“运营驾驶舱”。

---

## 2. 基于 GitHub 代码的前端改造边界

### 2.1 技术栈判断

项目说明中前端为 Vue3 + Element Plus，并以静态资源形式挂载到 FastAPI 后端。这意味着本次改造可以优先控制在前端层，不需要调整业务接口。已有页面使用 Element Plus、静态 HTML/CSS/JS 和接口数据加载方式。

### 2.2 已有接口继续复用

当前前端已有的数据加载逻辑主要使用：

```text
/stats/summary
/stats/dimensions
/health
/logs
/config/*
```

因此本轮 UI 改造建议不修改后端，只做：

1. API 数据到 ViewModel 的前端适配；
2. 组件拆分；
3. CSS 变量和布局重构；
4. 菜单去重和前端配置治理；
5. 图表展示形式优化；
6. 局部 loading/error 状态增强。

## 6. 分阶段实施计划

## 第一阶段：低风险样式优化

目标：不动接口，不动业务逻辑，只改样式和结构。

修改内容：

1. 新增 CSS 变量。
2. 侧边栏宽度从当前约 210px 调整到 240px。
3. 左侧菜单字体增大，菜单项高度调整为 40px。
4. 主内容背景改为浅灰蓝。
5. 卡片统一圆角、阴影、边框。
6. KPI 卡片统一三层结构：标题、主数值、辅助说明。
7. 页面主内容增加最大宽度和统一间距。

验收标准：

1. 首页功能不变。
2. 原接口不变。
3. 原有菜单点击仍可跳转。
4. 页面视觉明显更清晰。

---

## 第二阶段：菜单治理

目标：解决截图中菜单重复和滚动过长问题。

修改内容：

1. 抽离菜单配置为 `menuConfig`。
2. 按“工作台、质控业务、推送管理、系统运维、配置中心”重组。
3. 去除重复分组。
4. 当前选中菜单增加蓝色背景和左侧高亮条。
5. 支持菜单折叠。
6. 可选：根据角色隐藏无权限菜单。

验收标准：

1. 同一功能入口只出现一次。
2. 左侧菜单无需长时间滚动即可找到核心功能。
3. 首页总览、患者质控、前置机告警、质控闭环、推送日志、系统配置均可正常进入。

---

## 第三阶段：驾驶舱布局重排

目标：将首页从“功能堆叠”调整为“运营驾驶舱”。

修改内容：

1. 增加顶部 Banner。
2. KPI 指标分为“今日运行指标”和“风险闭环指标”。
3. 趋势图和风险等级图形成主视觉区。
4. 质控维度分布改为横向条形图。
5. 系统健康矩阵改为服务状态列表。
6. 前置机闭环改为步骤/漏斗展示。

验收标准：

1. 第一屏能看到系统状态、今日推送、今日不一致、前置机失败。
2. 高危、未查看、待处理等风险指标更突出。
3. 图表可读性提升，中文标签不再大面积倾斜拥挤。

---

## 第四阶段：交互增强

目标：提高实际工作效率。

修改内容：

1. 指标卡片支持点击跳转到明细页面。
2. 图表支持点击筛选。
3. 最近前置机告警点击进入详情。
4. 每个模块独立 loading 和 error。
5. 增加最近刷新时间。
6. 刷新按钮增加 loading 状态。
7. 可选：增加自动刷新开关。

验收标准：

1. 单个模块接口失败不影响整个首页。
2. 用户可从首页快速进入失败日志、未查看、风险明细。
3. 刷新体验明确，不出现无反馈等待。

## 8. 推荐优先级

### P1：马上做

1. 左侧菜单去重。
2. 侧边栏字号和宽度调整。
3. 首页卡片样式统一。
4. KPI 指标分组。
5. 系统健康列表优化。

### P2：本轮一起做

1. 质控维度横向条形图。
2. 前置机闭环步骤式展示。
3. 每个模块独立 loading/error。
4. 指标卡片点击跳转。

### P3：后续优化

1. 自动刷新开关。
2. 菜单搜索。
3. 用户收藏常用功能。
4. 多主题或夜间模式。
5. 移动端适配增强。

---

## 9. 注意事项

1. 不要为了 UI 好看改后端字段。
2. 不要删除现有接口调用。
3. 菜单去重时必须确认路由仍然可达。
4. 图表改造时注意空数据、异常数据。
5. “未知科室”要标注为数据映射待完善，避免误认为真实科室。
6. 所有数值颜色需要有业务阈值，不能只为了好看随意配色。
