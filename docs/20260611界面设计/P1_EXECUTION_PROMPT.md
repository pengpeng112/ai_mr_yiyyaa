# 执行提示词：Med-Audit 前端 P1 界面改造

## 项目背景

- 项目：`pengpeng112/ai_mr_yiyyaa`（路径 `F:\python\前后端代码\ai_mrzk`）
- 技术栈：FastAPI 后端 + Vue3 + Element Plus 静态前端
- 前端资源位于 `static/` 目录：`templates/pages/`（HTML 页面片段）、`scripts/modules/`（JS 模块）、`styles/pages/`（页面样式）
- 所有页面通过 `static/index.html` 中的 Vue 应用统一挂载，`activeMenu` 控制显示
- 侧边栏菜单配置在 `static/scripts/navigation.js`
- 请先阅读 `AGENTS.md` 和 `docs/20260611界面设计/_compliance_check.md` 了解背景与约束

## 已确认的设计决策（必须遵守）

1. **前置机告警** 从"患者质控"页面的 Tab 拆分为 **独立菜单页面**，放入左侧菜单"质控业务"组。
2. **推送进度** 作为 **新增独立页面**，放入左侧菜单"推送管理"组。
3. **权限管理** 保留在"运维管理"组。
4. **整改闭环率等 KPI** 优先通过前端已有数据计算展示；如需新增后端接口，必须先请示。
5. 本次改造以 **P1 代码改造** 为主，只做前端展示层调整，不修改后端业务逻辑。

## 通用约束（全程遵守）

1. **只做前端**：不修改后端接口、业务逻辑、数据库模型。
2. **如需新增后端 KPI 接口**：先停止并请示用户，不要自行实现。
3. **保留现有功能**：所有原有按钮、接口调用、状态字段必须继续可用。
4. **命名与路由**：
   - 新页面 HTML 文件名使用小写、连字符风格，如 `relay_alert.html`、`push_progress.html`。
   - 菜单 key 与 `activeMenu` 保持一致。
5. **样式规范**：
   - 继续使用 CSS 变量（`--color-primary` 等）。
   - 侧边栏宽度保持 240px。
   - 卡片圆角 8px～12px。
   - 紧凑列表行高 38px～44px。
6. **测试验证**：
   - 每步完成后运行 `python -m compileall app tests scripts`。
   - 在浏览器中验证菜单切换、页面加载、筛选、分页、弹窗/抽屉正常。
   - 无 JS 报错、无 404。
7. **Git 提交**：
   - 每完成一个 P1 步骤单独提交，commit message 格式：`feat: P1-步骤X-简短说明`。
   - 不要一次提交大量无关文件。
   - 不要推送，除非用户明确要求。
8. **部署**：
   - 如需部署到生产容器 `med-audit`（`10.10.8.84:40022`，root / `P@ssw0rd@123`），使用 `docker cp` + `docker commit -m "msg" med-audit med-audit:latest` + `docker restart med-audit` 模式。
   - 不要执行 `docker-compose down`。
   - 部署前确认已通过 compileall 和本地功能验证。
