# 患者质控界面紧凑版前端设计方案

> 页面：患者质控  
> 原则：只改前端，不改后端接口；保留原有日志、报告、弹窗、详情、分页、筛选等功能；优先保证患者列表一屏展示更多记录。

## 1. 设计定位

> **已确认（2026-06-14）**：前置机告警从患者质控页面拆分为独立菜单页面，患者质控页面仅保留“患者总览”。

患者质控页面不是展示型驾驶舱，而是高频工作列表页，建议定位为：

```text
紧凑型患者质控工作台
```

核心目标：

```text
1. 一屏显示更多患者
2. 筛选条件一行完成
3. 患者风险、状态、医生查看、得分一眼可扫
4. 右侧显示当前选中患者摘要，避免频繁打开弹窗
5. 原有质控报告、弹出提醒、详情查看功能全部保留
```

## 2. 基于现有代码的改造边界

当前仓库前端是静态 Vue + Element Plus，主页面在 `backend/static/index.html` 中，已有 flex 布局、左侧菜单、顶部标题和内容区。当前代码中已有 `logsList`、`logsTotal`、`logsPage`、`logsFilter`、`logsLoading`，以及 `viewLogDetail(row)`、`openReport(logId)`、`openPopup(logId)` 等功能。

如果患者质控页面暂时没有独立接口，可以先复用日志列表数据作为患者维度视图；后续再接专用患者质控接口。

## 3. 页面结构

```text
顶部条 48px
├─ 页面标题：患者质控
└─ 用户、时间、系统状态、退出

操作条 46px
├─ 标题 + 简短说明
└─ 刷新 / 导出列表 / 批量提醒 / 生成汇总

统计细条 38px
├─ 患者总数
├─ 不一致
├─ 高危
├─ 未查看
├─ 待反馈
├─ 已闭环
└─ 今日新增

筛选条 44px
├─ 风险等级
├─ 质控状态
├─ 在院科室
├─ 出院科室
├─ 姓名
├─ 患者ID
├─ 住院号
├─ 核查类型
├─ 日期范围
├─ 查询
└─ 重置

主体区
├─ 左侧：患者质控列表
└─ 右侧：选中患者质控摘要
```

## 4. 紧凑尺寸规范

```text

顶部栏高度：48px
操作条高度：46px
统计条高度：38px
筛选条高度：44px
表头高度：34px
表格行高：40px
表格字体：12px～13px
右侧摘要宽度：320px
左侧菜单宽度：224px
```

## 6. 右侧患者摘要

右侧摘要不替代详情页，只做快速预览，建议包含：

```text
1. 基本信息
2. 质控结果
3. 主要问题
4. 维度状态
5. 处理记录
6. 快捷操作：打开报告、提醒医生、查看文书、闭环反馈
```

## 7. 不影响功能的实现要求

1. 保留原有菜单、路由、接口和函数。
2. 如果没有患者质控专用接口，先基于 `logsList` 适配显示。
3. `报告` 按钮继续调用 `openReport(row.id)`。
4. `更多` 下拉中保留：详情、弹出提醒、复制患者ID、加入闭环。
5. 点击表格行只更新右侧摘要，不直接触发后端写操作。
6. 批量提醒必须二次确认。
7. 导出列表优先复用现有导出逻辑；没有接口时先前端导出当前筛选结果。

## 9. 回归测试

1. 患者质控页面正常打开。
2. 筛选条件可输入和重置。
3. 患者列表可正常滚动。
4. 点击患者行右侧摘要更新。
5. 报告按钮可正常打开报告。
6. 更多菜单中的详情、弹出提醒可用。
7. 批量提醒、导出按钮不影响原有功能。
8. 小屏下右侧摘要下移，列表仍可横向滚动。
9. 后端接口失败时页面不崩溃。
10. 不影响推送日志、首页总览、配置管理等其他页面。
